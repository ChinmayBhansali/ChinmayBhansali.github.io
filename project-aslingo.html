Component 1

The Machine Learning Pipeline

A custom end-to-end pipeline built to act as the intelligence behind the gesture recognition.

Data Collection

An automated script prompts users to perform specific signs (e.g., alphabet, "hello"). It utilizes OpenCV to capture 20 rapid frames per sign directly from the webcam, generating a structured visual dataset.

Feature Extraction

Instead of raw pixels, Google's MediaPipe Holistic model extracts exact 3D spatial coordinates (landmarks) for the user's pose (33 points) and hands (21 points each). We isolated only hand landmarks to reduce noise and optimize accuracy.

Model Architecture

The arrays are fed into a Keras Sequential neural network. We used an MLP architecture (Flatten layer -> two Dense layers with 128 ReLU neurons -> Dense Softmax output layer), trained over 100 epochs using the Adam optimizer.

Component 2

Web Application & Real-Time Inference

Bridging the frontend user interface with the computationally intensive ML model.

Routing & State

A Flask server manages the entire application state. It serves various pages (Home, Learning, Translator, Lessons) and handles basic user authentication (Login/Register) utilizing a lightweight JSON-based data store.

The Translation Endpoint

The core interactive feature lives at /process_frame. The frontend JavaScript captures individual frames from the user's active webcam and transmits them to this endpoint as base64-encoded strings via rapid POST requests.

Live Inference Process

  • Base64 string is decoded and converted back into an OpenCV image array.
  • MediaPipe processes the live image to extract pose and hand landmarks, mirroring the extraction methodology used during training.
  • The pre-trained model.h5 evaluates this array, makes a prediction, and maps the highest probability index to the corresponding character.
  • The translated text is returned directly to the user's screen in real-time.
Reflection

Project Strengths & Future Expansion

Analyzing architectural decisions and identifying areas for growth beyond the hackathon environment.

Engineering Strengths

Low-Latency Processing

By utilizing MediaPipe for landmark extraction rather than passing raw high-resolution pixel data through an expensive Convolutional Neural Network (CNN), we drastically reduced the computational load. This allows for smooth, real-time inference in a standard web browser.

Modular ML Pipeline

Building a complete end-to-end pipeline—from scripting automated webcam data collection to feature extraction and model training—demonstrates a strong structural understanding of bridging raw, messy real-world data with predictive models.

Interactive Tooling

The project successfully applies complex backend processing to a tangible accessibility and educational problem, creating a highly interactive feedback loop that genuinely enhances the learning experience.

Future Expansion (Next Steps)

1
Temporal Dynamics (LSTM/RNN)

The current Multi-Layer Perceptron evaluates static individual frames. Transitioning to a Recurrent Neural Network (RNN) or Long Short-Term Memory (LSTM) architecture would empower the system to recognize complex, multi-stage ASL signs that rely on continuous fluid movement over time, rather than just isolated hand shapes.

2
Robust Security

Upgrading the hackathon-friendly plaintext JSON authentication to a secure, scalable database (such as PostgreSQL) utilizing hashed passwords (e.g., bcrypt) to protect user accounts and progress tracking securely and efficiently.
